# IIMS 课程编写工作流

> 目标：把 30 课时课程写成可教学、可实操、可面试复盘的就业导向材料。每一课都必须基于本项目真实源码，而不是泛泛讲技术名词。

## 1. 文件命名

课程文件统一放在：

```text
C:\Users\MoLin\Desktop\IIMS\jiaoxue
```

命名规则：

```text
lesson-01-project-onboarding.md
lesson-02-dev-environment.md
...
lesson-30-resume-interview.md
```

## 2. 每课固定结构

每一课至少包含：

1. 课程定位
2. 教学目标
3. 就业目标
4. 涉及源码文件
5. 核心概念
6. 源码链路
7. 实操步骤
8. 常见错误
9. 课堂演示脚本
10. 学生练习
11. 验收标准
12. 作业
13. 面试表达
14. 最终交付物

## 3. 源码研究规则

写每一课前必须至少确认：

- 后端 Controller / Service / Mapper / XML 位置。
- 前端 views / api / utils / store / router 位置。
- 数据库表或配置项位置。
- 如果是部署课，要确认脚本、配置、端口和目录。

## 4. 深度标准

每课不能只写“做什么”，还要写：

- 为什么要这么做。
- 如果不这么做会怎样。
- 失败后如何排查。
- 这件事在面试中怎么讲。
