# 第 30 课：二次开发、简历包装与面试攻防
> 课程定位：最后一课把项目从“我能跑起来”变成“我能讲清楚、能二次开发、能应对追问”。目标不是背话术，而是把真实源码、部署过程、排障经历整理成可信的求职表达。

## 1. 本课目标

学完本课后，学生应该能做到：

1. 提炼 IIMS 项目的技术栈。
2. 提炼 3 到 5 个可写进简历的项目亮点。
3. 准备一套可演示的功能路径。
4. 准备 Spring Boot、MyBatis Plus、Redis、MinIO、Spring AI、RAG 的面试回答。
5. 设计一个二次开发任务，证明自己不是只会启动项目。
6. 形成最终求职项目交付包。

## 4. 简历项目描述

示例：

```text
基于 Spring Boot 3、Vue 3 和 Spring AI 开发的智能信息管理平台，包含用户权限、档案管理、知识库管理、文件上传、AI 流式对话和 RAG 知识库问答等功能。系统支持模型配置动态化，可接入 OpenAI 兼容接口、DeepSeek 和 Ollama，并使用 Milvus 实现向量检索。
```

注意：

```text

```

更稳妥的表达：

```text
参与项目部署、源码梳理、功能调试和二次开发。
```

如果你后续真的改了功能，再写具体开发内容。

## 5. 简历亮点写法

### 5.1 登录鉴权

```text
基于 Sa-Token 实现登录鉴权和接口权限控制，结合角色、菜单、权限标识完成后台 RBAC 权限管理。
```

### 5.2 数据访问

```text
使用 MyBatis Plus 完成通用 CRUD，复杂查询通过 XML Mapper 编写动态 SQL，并结合 PageHelper 实现分页查询。
```

### 5.3 文件服务

```text
接入 MinIO 对象存储，实现文件上传、MD5 秒传、短链访问和文件元数据管理。
```

### 5.4 AI 对话

```text
基于 Spring AI 动态构建 ChatModel，支持 DeepSeek、OpenAI 兼容接口和 Ollama，并通过 SSE 实现大模型流式响应。
```

### 5.5 RAG 知识库

```text
使用 EmbeddingModel 将知识库文档向量化写入 Milvus，提问时按知识库范围检索 topK 相关片段，并结合 Prompt 实现 RAG 问答。
```

### 5.6 部署排障

```text
完成阿里云 ECS 部署，使用 Nginx 托管前端静态资源，后端以 Spring Boot jar 运行，并通过 Docker 部署 MySQL、Redis、MinIO 等中间件。
```

## 6. 推荐写进简历的 4 条

如果简历空间有限，推荐写：

```text
1. 负责项目本地与云服务器部署，完成 MySQL、Redis、MinIO 等中间件配置和初始化 SQL 修复。
2. 梳理并调试 Sa-Token 登录鉴权、RBAC 菜单权限和用户角色关系，保证后台接口访问控制正常。
3. 接入 Spring AI 模型配置链路，支持 DeepSeek、OpenAI 兼容接口和 Ollama，通过 SSE 实现 AI 流式对话。
4. 基于知识库、EmbeddingModel 和 Milvus 实现 RAG 检索增强问答，支持按知识库范围检索 topK 文档片段。
```

这些表达既能覆盖项目核心，也比较好展开讲。

## 8. 必备面试问题

### 8.1 你这个项目用了哪些技术？

回答：

```text
后端使用 Spring Boot 3 和 Java 17，数据访问使用 MyBatis Plus 和 XML Mapper，权限使用 Sa-Token，缓存使用 Redis，文件存储使用 MinIO。AI 部分使用 Spring AI 接入多种模型，通过 SSE 实现流式对话，并结合 EmbeddingModel 和 Milvus 实现知识库 RAG 问答。前端使用 Vue 3、Vite、TypeScript 和 Element Plus。
```

### 8.2 你负责了什么？

稳妥回答：

```text
我主要负责项目部署、源码梳理、环境配置、运行问题修复和 AI 模块联调。具体包括数据库初始化脚本修复、前后端启动配置、服务器部署、模型配置、SSE 对话调试，以及知识库向量化链路梳理。后续我也基于这个项目做二次开发练习。
```

### 8.3 Spring AI 在项目里怎么用？

回答：

```text
项目把模型配置存在数据库中，后端根据模型 id 查询模型类型、base URL、模型名和 API Key，然后动态构建 Spring AI 的 ChatModel。对话时把历史消息、用户问题和知识库检索结果组装成 Prompt，调用 ChatModel.stream 获取流式输出，再通过 SSE 推给前端。
```

### 8.4 RAG 怎么实现？

回答：

```text
知识库文章先通过 EmbeddingModel 转成向量并写入 Milvus。用户提问时，系统把问题也转成向量，在 Milvus 中按 wikiId 过滤并检索 topK 相关片段。然后把检索结果和用户问题一起组装成 Prompt，让 ChatModel 基于知识库内容生成回答。
```

### 8.5 为什么需要 Redis？

回答：

```text
Redis 在后台系统中通常用于缓存、会话、验证码、热点数据或分布式状态。具体到这个项目，需要结合实际配置和代码说明使用点；如果面试官追问，我会从配置类、缓存注解和业务调用位置定位它的实际职责。
```

这个回答诚实，不硬编。

### 8.6 MyBatis Plus 和 XML Mapper 怎么分工？

回答：

```text
简单单表 CRUD 使用 MyBatis Plus 提高开发效率，复杂关联查询、动态条件、权限过滤和分页列表则放到 XML Mapper 中手写 SQL。比如用户角色菜单这类 JSON 字段和多表关系查询，用 XML 更直观也更容易优化。
```

### 8.7 SSE 和 WebSocket 为什么选 SSE？

回答：

```text
AI 对话场景主要是服务端向客户端持续推送生成内容，客户端不需要高频双向通信，所以 SSE 更轻量，基于 HTTP，前端处理也简单。WebSocket 更适合强双向实时交互，比如聊天房间或协同编辑。
```

## 9. 二次开发建议

为了让项目更像自己的，建议做至少一个二次开发：

### 9.1 模型连通性检测

新增“检测模型”按钮：

```text
点击后后端真实调用模型服务。
返回是否可用、耗时、错误信息。
```

价值：

```text
非常贴合当前项目痛点。
面试很好讲。
```

### 9.2 AI 对话引用来源

RAG 回答后展示引用文档：

```text
显示知识库名称、文章标题、相似度。
```

价值：

```text
提升可信度。
体现 RAG 产品思维。
```

### 9.3 知识库向量化进度

显示：

```text

成功

```

价值：

```text
把异步任务做成可观测功能。
```

展示：

## 10. 最推荐二开任务

推荐优先做：

```text
模型连通性检测
```

原因：

- 范围适中。
- 和当前模型配置痛点强相关。
- 前后端都能改到。
- 面试时可以清楚讲技术链路。

实现思路：

```text
前端模型列表增加检测按钮。
后端新增 /model/check/{id} 接口。
Service 根据模型 id 构建 ChatModel 或 EmbeddingModel。
发送一条最小测试请求。
返回成功、耗时、错误信息。
```

不要写：

```text
独立设计并开发完整系统
负责全部架构
高并发百万级
自研大模型
```

可以写：

```text

```

真实可讲，比夸张更安全。

## 12. 面试被追问怎么办

如果问到没掌握的地方，不要硬编。

推荐结构：

```text
这个点我目前主要掌握到项目使用层面。
源码里我定位到相关位置是……
我的理解是……
如果继续优化，我会从……入手。
```

例如 Redis：

```text
这个项目里 Redis 的具体业务使用点我会先从配置和缓存调用定位。我目前能确定它作为中间件被项目依赖，后续如果要优化，会重点看登录态、缓存注解和热点数据是否使用 Redis。
```
